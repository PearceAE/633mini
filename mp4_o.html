I copied each of my previous mini projects before I ran them through the validator. Luckily I only had a few errors for each of my Mini Projects except for Mini Project 3, which was due to the audio file. I corrected the corresponding errors which usually dealt with a closing tag or an unidentified attribute. I did receive the Byte-Order Mark found in UTF-8 file warning and followed the steps outlined in Brightspace to correct the issue but was given this feedback information "Sorry, this type of URL scheme ("file") is not supported by this service. Please check that you entered the URL correctly." I instead validated by direct input and received this message "Unlike the “by URI” and “by File Upload” modes, the “Direct Input” mode of the validator provides validated content in the form of characters pasted or typed in the validator's form field. This will automatically make the data UTF-8, and therefore the validator does not need to determine the character encoding of your document, and will ignore any charset information specified. If you notice a discrepancy in detected character encoding between the “Direct Input” mode and other validator modes, this is likely to be the reason. It is neither a bug in the validator, nor in your document." Based on this feedback from the validator, I believed this indicated that my projects passed, I then verified they were able to open in Mozilla Foxfire and Google Chrome.
